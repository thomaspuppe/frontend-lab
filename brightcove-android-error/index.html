Brightcove Android Error

VIDEOJS: ERROR: (CODE:4 MEDIA_ERR_SRC_NOT_SUPPORTED) The media could not be loaded, either because the server or network failed or because the format is not supported.

This test case has no autostart and has advertising disabled (Brightcove Player).

You can also find it in production with autostart and ads: http://www.zeit.de/video/index (Brightcove Player). The first video (in doubt, search for the element '.video-stage-main') has this issue. On page load, there is a placeholder image. On click it gets replaced by an iframe, which then has the error.

We have tried multiple hypotheses

  • Autostart on/off: makes no difference.
  • For a while we thought the ads would trigger the error. But now we have disabled ads in the player, and still have the same error.

Some more information

  • IMPORTANT: There are videos in our which do work on the same setup: old-video.html
  • I have created a test case which uses the same player, but has ads: with-ads.html
  • Even when the error occurs, the network traffic shows that the m3u8 file and a working ts file get transmitted via the network. Which means, that the part "or because the format is not supported" of our error message might be the important one.
  • Other Brightcove Customers with similar setup have working videos.
